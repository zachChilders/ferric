# Task: M1 VM Implementation (Tree-Walker)

## Objective
Implement the `ferric_vm` crate with a tree-walking interpreter that executes the AST directly. Implements the `Executor` trait to enable future VM replacements.

## Architecture Context
- The VM is the execution stage of the pipeline
- It must expose the `Executor` trait and `TreeWalker` implementation
- This tree-walker will be replaced in M3 with a bytecode VM
- Both will implement the same `Executor` trait, making the replacement transparent

## Feature Requirements

### Environment (Call Stack)
Track local variables and their values in an environment stack.

```rust
struct TreeWalker {
    env_stack: Vec<HashMap<DefId, Value>>,
    ast: ParseResult,  // cached for function lookups
    resolve: ResolveResult,
    natives: NativeRegistry,
}
```

### Execution Flow
1. **Function calls:**
   - Look up the function definition in the AST
   - Push a new environment frame
   - Bind parameters to argument values
   - Execute the function body
   - Pop the environment frame
   - Return the result

2. **Let bindings:**
   - Evaluate the initializer expression
   - Store the value in the current environment frame

3. **Variable lookup:**
   - Use the `resolve.resolutions` map to get the DefId
   - Look up the DefId in the environment stack (innermost to outermost)

4. **Binary operations:**
   - Evaluate both operands
   - Apply the operator based on value types
   - Return the result

5. **If expressions:**
   - Evaluate the condition
   - If true, evaluate then-branch
   - If false, evaluate else-branch (or return Unit)

6. **Blocks:**
   - Push a new environment frame
   - Execute all statements
   - Evaluate the final expression (if any)
   - Pop the environment frame
   - Return the final value

7. **Return statements:**
   - Evaluate the expression
   - Throw an early-return exception (or use Result)

### Native Function Support
Native functions are provided via `NativeRegistry`:

```rust
// In ferric_stdlib
pub struct NativeRegistry {
    functions: HashMap<Symbol, NativeFn>,
}

pub type NativeFn = fn(&[Value]) -> Result<Value, String>;

impl NativeRegistry {
    pub fn new() -> Self;
    pub fn register(&mut self, name: Symbol, f: NativeFn);
    pub fn get(&self, name: Symbol) -> Option<&NativeFn>;
}
```

When calling a function, check if it's native first:
```rust
if let Some(native) = natives.get(name) {
    return native(&args);
} else {
    // User-defined function - look up in AST
}
```

## Implementation Notes

### TreeWalker Structure
```rust
struct TreeWalker {
    env_stack: Vec<HashMap<DefId, Value>>,
    current_items: Vec<Item>,  // cached AST for function lookups
    current_resolve: ResolveResult,
    natives: NativeRegistry,
}

impl TreeWalker {
    fn new() -> Self;

    fn eval_item(&mut self, item: &Item) -> Result<Value, RuntimeError>;
    fn eval_stmt(&mut self, stmt: &Stmt) -> Result<(), RuntimeError>;
    fn eval_expr(&mut self, expr: &Expr) -> Result<Value, RuntimeError>;

    fn push_env(&mut self);
    fn pop_env(&mut self);
    fn define(&mut self, def_id: DefId, value: Value);
    fn lookup(&self, def_id: DefId) -> Result<&Value, RuntimeError>;
}
```

## Critical Rules to Enforce

### Rule 6 - VM is behind a trait
Never call `TreeWalker` directly. Always use `Executor` trait.
This makes replacing it with `BytecodeVM` in M3 transparent.

### Rule 7 - Value is never constructed directly outside ferric_vm
Other crates use `Value::new_int(5)`, never `Value::Int(5)`.
This enables swapping the value representation later (e.g., NaN-boxing).

### Rule 5 - Every error carries a Span
All `RuntimeError` variants must include a `Span`.
