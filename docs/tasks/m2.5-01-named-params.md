# M2.5 — Task 1: Named Parameters

> **Do this task first.** It changes `CallExpr` in `ferric_common`, which is read by
> every downstream stage. The shape of `CallExpr` must be settled before any other
> M2.5 task begins. Tasks 2, 3, and 4 may proceed in any order after this one is
> complete and all tests are passing.

---

## What this task does

Makes named parameters mandatory at all Ferric call sites. Positional call syntax
becomes a compile error. Parameters may have default values. Arguments at the call
site may be provided in any order — the resolver canonicalises to definition order
before the type checker and VM see them.

---

## ferric_common — changes required

### 1. Add `NamedArg`

```rust
pub struct NamedArg {
    pub span:  Span,
    pub name:  Symbol,
    pub value: Box<Expr>,
}
```

### 2. Update `CallExpr`

Replace `args: Vec<Expr>` with `args: Vec<NamedArg>`:

```rust
// Before
pub struct CallExpr {
    pub span:   Span,
    pub callee: Box<Expr>,
    pub args:   Vec<Expr>,
}

// After
pub struct CallExpr {
    pub span:   Span,
    pub callee: Box<Expr>,
    pub args:   Vec<NamedArg>,      // named, any order at call site
}
```

### 3. Update function parameter definitions

Add `default` to parameter nodes so the parser can record default values:

```rust
// Before
pub struct Param {
    pub span: Span,
    pub name: Symbol,
    pub ty:   TypeExpr,
}

// After
pub struct Param {
    pub span:    Span,
    pub name:    Symbol,
    pub ty:      TypeExpr,
    pub default: Option<Box<Expr>>,
}
```

`default` is `None` for all existing parameters. This is a purely additive change
to the struct — existing code constructing `Param` must add `default: None`.

## Stage changes

### Lexer — no new tokens

`:` already exists. No changes needed.

### Parser

**At call sites:** parse the argument list as `NamedArg` sequences. If an argument
is present without a `name:` prefix, emit `ParseError::PositionalArg` with the
span of the offending argument and continue parsing (accumulate, don't abort).

```
// LEGAL
connect(host: "localhost", port: 8080, timeout: 30)

// LEGAL — any order
connect(timeout: 30, host: "localhost", port: 8080)

// ILLEGAL — emit ParseError::PositionalArg
connect("localhost", 8080, 30)
```

**At function definitions:** parse optional `= <expr>` after the type annotation
to populate `Param::default`.

```rust
fn connect(host: Str, port: Int, timeout: Int = 30) -> Bool { ... }
```

**New error variant:**

```rust
// Add to ParseError:
ParseError::PositionalArg { span: Span }
```

Carry `Span` (Rule 5).

### Name Resolution

Two new jobs:

1. **Validate completeness.** For each call, check that every non-default parameter
   has a corresponding `NamedArg`. Emit `ResolveError::MissingArg { param: Symbol, span: Span }`
   for each missing required parameter.

2. **Validate names.** Check that every `NamedArg::name` matches a parameter name
   in the callee's definition. Emit `ResolveError::UnknownArg { name: Symbol, span: Span }`
   for unrecognised names.

3. **Canonicalise order.** Reorder `CallExpr::args` to match the callee's parameter
   definition order, inserting evaluated defaults for omitted optional parameters.
   After this pass, all downstream stages (type checker, VM) see args in definition
   order and do not need to know about named parameters at all.

**New error variants:**

```rust
// Add to ResolveError:
ResolveError::MissingArg   { param: Symbol, call_span: Span }
ResolveError::UnknownArg   { name: Symbol,  span: Span      }
```

Both carry `Span` (Rule 5).

### Type Checker

No structural change. After resolver canonicalisation, args are in definition order.
Type-check `args[i].value` against `params[i].ty` as before. The fact that args
were originally named is invisible at this stage.

### VM (TreeWalker)

No structural change. Args are in definition order after resolver canonicalisation.
Evaluate and bind to slots as before.

## Migration: existing test programs

All M1 and M2 test programs must be updated to use named-parameter call syntax.
This includes calls to stdlib functions.

**Stdlib function signatures to update:**

```rust
// Before / After — same behaviour, named params added
println(s: Str)
print(s: Str)
int_to_str(n: Int) -> Str
float_to_str(n: Float) -> Str
bool_to_str(b: Bool) -> Str
int_to_float(n: Int) -> Float
```

**Example migration:**

```rust
// M1/M2 (old)
println(message)
println(fibonacci(10))

// M2.5 (new)
println(s: message)
println(s: fibonacci(n: 10))
```

Update every call site in the test suite. The parser will catch any missed sites
with `ParseError::PositionalArg`.
